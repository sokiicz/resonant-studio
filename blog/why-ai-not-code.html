Why I'm Building Apps with AI
Instead of Learning to Code

The story of how a digital marketer with too many ideas discovered they could ship real software without ever touching a code editor — and why that changes everything.

I've always had ideas. That's never been the problem. The problem was always the gap between "I wish this existed" and actually making it exist.

For years, that gap was filled by either hiring someone, convincing a developer friend to help, or simply letting the idea die. Learning to code myself always felt like a multi-year commitment — and honestly, it wasn't the code I wanted. It was the thing the code would make.

I'm a digital marketer. I spend my days thinking about audiences, messages, user journeys, what people need. I have a good sense of what makes something useful and what makes it feel right. What I've never had is the ability to build it myself.

Then Claude Code happened

I started using Claude Code on a whim. I had a small web app idea — nothing complex, just a tool I wanted for myself. I described it, conversationally, like I was explaining it to a friend who happened to be a brilliant developer.

An hour later, the app existed.

Not a prototype. Not a wireframe. A working, styled, functional application. And I hadn't typed a single line of code.

"The most valuable skill in the AI era isn't coding — it's knowing what you want to build, and being able to describe it clearly."

What actually changed

The bottleneck in my creative process wasn't ideas. It was implementation. Claude Code removed that bottleneck entirely. Now when I have an idea, the question is no longer "can I build this?" — it's always yes. The question becomes "should I build this? Does it solve a real problem? Will it be useful?"

That's a much more interesting question to spend your energy on.

And here's the thing: as a digital marketer, I'm actually well placed to answer it. I understand audiences. I understand user needs. I understand what makes something worth using. I was already good at the part that matters most — I just couldn't execute. Now I can.

The open source decision

When I started shipping apps, I faced a question I hadn't anticipated: should I charge for them? The honest answer was: no. Not because the apps aren't valuable, but because the value came from the AI that wrote them. Charging felt like claiming credit I hadn't earned.

So I made everything free and open source. Every app on Resonant Labs is available to use, fork, learn from, or build upon. No strings attached.

What I want you to take away

If you're reading this and you have ideas but no coding skills — this is your moment. The tools exist. The barrier is lower than it's ever been. You don't need to spend two years learning JavaScript. You need to learn how to describe what you want, clearly and with specificity.

That's it. That's the new skill. And you probably already have most of it.

Try Claude Code. Build something small. See what happens.
